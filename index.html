Purples

2020.01.01 ~ 04.12 / 2명 / 1차 완성
팀프로젝트 SNS react next redux-toolkit ant design

1. 프로젝트 기획, 디자인

감성적인 sns를 만들고 싶다는 생각을 했고, react 프로젝트로 sns를 만들자는 제안을 했습니다.
감성적인 느낌을 표현하고 싶어 보라색을 테마로 정하고, purple과 사람들(s)을 조합해서 purples로 정하게 되었습니다.
디자인은 figma로 진행했습니다.

2. 전체 페이지와 컴포넌트 분리, 작성

각 페이지들을 나누고 공통 레이아웃을 잡아서 styled components로 스타일링을 했으며,
페이지에 필요한 컴포넌트들을 적당히 분리해서 작성했습니다.

3. redux-toolkit 으로 전환

redux와 redux-saga로 상태관리를 하게되었는데, 하다보니 코드가 불필요하게 너무 많다는 것이 느껴졌습니다.
그래서 코드를 줄이기 위해 고민하던 중 toolkit을 보게 되었는데 코드가 훨씬 깔끔한게 보였습니다.
그렇게 redux와 redux-saga로 만들었던 모든 코드를 redux-toolkit으로 전환하는 작업을 했습니다.
정보가 들어오는 방식이 달라 애를 먹었지만 차분히 하나하나 확인해보며 무사히 전환을 완료했습니다.

4. 서버사이드 렌더링 적용

게시글 상세페이지로 갈 때, 미리 페이지 정보를 로드할 수 있게 serversideProps를 이용했습니다.
그리고 url로 접속시 next로 인해 로그인이 계속 풀리는 것을 알게되었고,
페이지마다 유저 정보를 가져올 수 있도록 dispatch했습니다.

5. 오류, 개선사항 노트 작성

프로젝트를 진행하며 오류나 개선사항에 대해 말로 하기보다는 분류해서 같이 볼 수 있다면 좋겠다는 생각이 들어
노션에 정리하여 공유하며 문제를 해결하려 했습니다.

6. 주도적으로 업그레이드 예정

좀 더 세련된 UI,UX를 제공하고 싶었는데 생각보다 어려웠습니다.
같이 프로젝트를 진행한 분과 틈틈이 계속 수정할 예정입니다.

포트폴리오 사이트

2020.04.01 ~ 04.13 / 혼자
포트폴리오 html / css javascript 스크롤

1. 마크업에 공을 들인 포트폴리오

article 태그를 적절히 섞어 div의 남발을 최대한 줄이기 위해 노력했습니다.
class 네이밍도 알아보기 쉽게 신경써서 작성하였고, css도 알아보고 수정하기 쉽게 하기위해 디테일을 줄여서 작성했습니다.

2. 탭, 메뉴 슬라이드 구현

css와 javascript를 이용하여 탭 기능과 메뉴의 슬라이드를 구현했습니다.
탭을 구현할 때는 클릭한 번호를 가져와서 배열로 저장한 탭에 active 클래스를 추가했습니다.
클릭한 메뉴로 이동을 할 때는 저장되었던 영역의 top값으로 이동(3번 참고)하도록 했습니다.
또 메뉴 슬라이드가 들어갈 때는 검정배경 영역 비활성화시 드래그가 안되는 경우를 확인해서, visibility 속성을 추가했습니다.

3. 스크롤, 인터렉티브 효과

영역별로 인터렉티브 효과를 주고, 메뉴에서 클릭시 각 영역으로 이동하기 위해 각 영역의 위치(top)와 높이(height)를 알아야 했습니다.
또 디바이스별로 높이가 다르기 때문에 reset이 될 때마다 값을 체크해야 했습니다.
영역별로 top과 height를 저장하고, 각 영역마다 인터렉티브 효과를 적용할 element들을 저장하도록 했습니다.
저장된 값들을 이용하여 스크롤의 위치를 파악하여 인터렉티브한 효과를 주고, 메뉴를 클릭시 해당영역으로 이동하게 됩니다.

2. 스로틀 적용

스크롤을 내릴 때 이벤트가 너무 많이 일어나는 것을 알았습니다.
스로틀을 이용하여 스크롤 이벤트를 일정시간 단위로 호출되도록 했습니다.

2. 모바일을 신경쓴 반응형

모바일에서도 편하게 볼 수 있으면 좋을 것 같다는 생각이 들어 먼저 모바일 영역에 맞춰 디자인을 하고 PC도 디자인 했습니다.
모바일용 css를 추가로 작성했습니다.
